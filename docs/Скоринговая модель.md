# Скоринговая модель

Документ фиксирует согласованную целевую модель критериев подбора программ для четырех сценариев:

- `С1` - подбор по свободному описанию.
- `С2` - подбор через AI-агента и пошаговую анкету.
- `С3` - углубленная траектория после пройденных программ.
- `С4` - траектория новых интересов после пройденных программ.

`Контекст пользователя` - внутренний набор критериев, который бот собирает из свободного текста, анкеты, ссылок на пройденные программы и уточнений пользователя. В него входят возраст, населенный пункт, организация, интересы, направленность, стоимость, расписание, ограничения, цели и пройденные программы.

| № | Критерий | Сценарии | Если совпал | Если не совпал | Если не указан / неизвестно | Таблицы и атрибуты |
|---:|---|---|---:|---:|---:|---|
| 1 | Населенный пункт программы | С1-С4 | 0, проходит | исключается до скоринга | подбор не запускается | `pfdo_programs.municipality_id`, `pfdo_main_municipalities.id/name`, контекст пользователя: населенный пункт |
| 2 | Ограничение по организации | С1-С4 | 0, проходит | исключается до скоринга | проходит | `pfdo_programs.organization_id`, `pfdo_programs.organization_name`, `pfdo_organizations.id/name`, контекст пользователя: организация |
| 3 | Возраст программы | С1-С4 | 0, проходит | исключается до скоринга | подбор не запускается | `pfdo_programs.age_group_min`, `pfdo_programs.age_group_max`, контекст пользователя: возраст |
| 4 | Стоимость группы или программы | С1-С4 | 0, если цена не выше указанной | исключается до скоринга | проходит | `pfdo_program_groups.period_price`; если нет, `pfdo_program_modules.price`, `pfdo_program_modules.normative_price`; контекст пользователя: стоимость |
| 5 | Исключение уже пройденных программ | С3, С4 | уже пройденные исключаются | 0 | не применяется | `pfdo_programs.id`, контекст пользователя: `completedProgramIds` |
| 6 | Форма обучения | С1-С4 | 0, проходит | исключается до скоринга | проходит | `pfdo_programs.edu_form`, `pfdo_program_education_forms.id/name`, контекст пользователя: форма обучения |
| 7 | Попадание в расписание | С1-С4 | +100 полностью, +50 частично | -100 | 0 | `pfdo_group_schedule_entries.week_day`, `pfdo_group_schedule_entries.start_time`, `pfdo_group_schedule_entries.end_time`, контекст пользователя: дни и время |
| 8 | Наличие мест / открыта запись | С1-С4 | +100, если есть места; +50, если мест нет, но запись открыта | -100, если запись закрыта | 0 | `pfdo_program_groups.free_places_counter`, `pfdo_programs.enrollment`, при необходимости `pfdo_program_groups.status` |
| 9 | Направленность программы | С1-С4 | +50 | -50 | 0 | `pfdo_programs.direction_id`, `pfdo_program_directions.id/name`, контекст пользователя: направленность |
| 10 | Размер группы | С1-С4 | +50 | -50 | 0 | `pfdo_program_modules.min_child_group`, `pfdo_program_modules.max_child_group`, контекст пользователя: желаемый размер группы |
| 11 | У программы есть темы из документов | С1-С4 | +10 | -100, если тем у программы нет | 0 | наличие строк в `pfdo_program_topic_aggregates` или `pfdo_program_calendar_topics` по `program_id` |
| 12 | Точное совпадение интереса с нормализованной темой | С1-С4 | +100 | 0 | 0 | `pfdo_program_topic_aggregates.normalized_topic_key`, `pfdo_program_topic_aggregates.normalized_topic_name`, контекст пользователя: явно указанные интересы |
| 13 | Совпадение интереса с категорией level 2 | С1-С4 | +70 | 0 | 0 | `pfdo_program_topic_classifications.category_code`, `pfdo_program_topic_classifications.category_name`, контекст пользователя: явно указанные интересы |
| 14 | Совпадение интереса с разделом level 1 | С1-С4 | +30 | 0 | 0 | `pfdo_program_topic_classifications.parent_code`, `pfdo_program_topic_classifications.parent_name`, контекст пользователя: явно указанные интересы |
| 15 | Fallback по названию, аннотации, ключевым словам | С1-С4 | +10 или +20, только если у программы нет нормализованных тем | 0 | 0 | `pfdo_programs.search_name`, `pfdo_programs.annotation_html`, `pfdo_programs.task_html`, `pfdo_program_keywords.name`, контекст пользователя: явно указанные интересы |
| 16 | Пользователь указал интерес, но нет тематического совпадения | С1-С4 | 0 | `score = 0`, кандидат отсекается | 0 | результат строк 12-15; применяется только если пользователь явно указал интерес |
| 17 | Совпадение с точной пройденной темой | С3 | +30 как тематическая связь | 0 | 0 | `pfdo_program_topic_aggregates.normalized_topic_key`, `pfdo_program_topic_aggregates.normalized_topic_name`, контекст пользователя: темы пройденных программ |
| 18 | Новая тема в той же категории level 2 | С3 | +70 | 0 | 0 | `pfdo_program_topic_classifications.category_code/name`, `pfdo_program_topic_aggregates.normalized_topic_key` |
| 19 | Новая тема в том же разделе level 1 | С3 | +40 | 0 | 0 | `pfdo_program_topic_classifications.parent_code/name` |
| 20 | Нет тематической связи с пройденным | С3 | 0 | `score = 0`, кандидат отсекается | 0 | результат строк 17-19 |
| 21 | Depth-сигнал углубления | С3 | +100 | 0 | 0 | `pfdo_programs.directory_level_id`, `pfdo_programs.search_name`, `pfdo_programs.annotation_html`, `pfdo_programs.task_html`, `pfdo_programs.age_group_min`, `pfdo_program_keywords.name`, `pfdo_program_topic_aggregates.hours_total`, `pfdo_program_topic_aggregates.hours_practice`, `pfdo_program_topic_aggregates.normalized_topic_name` |
| 22 | Нет depth-сигнала при простом повторе темы | С3 | 0 | -10 | 0 | результат строк 17-19 и 21 |
| 23 | Повтор изученной категории level 2 | С4 | `score = 0`, кандидат отсекается | 0 | 0 | `pfdo_program_topic_classifications.category_code/name`, контекст пользователя: темы пройденных программ |
| 24 | Новая категория level 2 в новом разделе level 1 | С4 | +100 | 0 | 0 | `pfdo_program_topic_classifications.category_code/name`, `pfdo_program_topic_classifications.parent_code/name` |
| 25 | Новая категория level 2 в том же разделе level 1 | С4 | +40 | 0 | 0 | `pfdo_program_topic_classifications.category_code/name`, `pfdo_program_topic_classifications.parent_code/name` |
| 26 | Любая содержательная тема по классификатору | С4 | +10 за тему, максимум +50 | 0 | 0 | `pfdo_program_topic_classifications.category_code/name`, `pfdo_program_topic_classifications.parent_code/name` |
| 27 | Уровень программы для новых интересов | С4 | +50, если `directory_level_id` в `1`, `2`, `4`, `5` | 0 | 0 | `pfdo_programs.directory_level_id`, `pfdo_program_levels.id/name` |

## Порог попадания в выдачу

После применения фильтров и начисления баллов программа попадает в выдачу только при итоговом `score > 0`. Если `score <= 0`, кандидат отсекается и не показывается пользователю.

## Правила расписания

- Выходные - суббота и воскресенье.
- Рабочие дни - понедельник-пятница.
- Утро - до `12:00`.
- День - `12:00-17:00`.
- Вечер - с `17:00`.

Полное совпадение расписания означает, что совпали и запрошенные дни, и запрошенный временной интервал. Частичное совпадение означает, что совпал только один из этих параметров.

## Правило fallback для интересов

Fallback по названию, аннотации и ключевым словам применяется только для программ, у которых нет данных в `pfdo_program_topic_aggregates` и `pfdo_program_topic_classifications`. Если тематические данные есть, интересы проверяются по нормализованным темам и классификатору.
