# Mattermost Interactive Menu Design

Date: 2026-06-17

Current runtime note: the production code now uses a text-only Mattermost UX. The bot sends numbered options and accepts numeric or text replies through the WebSocket transport. Slash commands, `MATTERMOST_ACTION_*`, and `MATTERMOST_SLASH_*` are historical design items and are not required by the current `src/index.js` runtime.

## Goal

Make Mattermost scenario entry behave close to Telegram while reusing the existing shared conversation runtime.

The feature adds two Mattermost-native UX layers:

- a custom slash command, `/traektoria`, with autocomplete in Mattermost;
- interactive message buttons for scenario choices and other inline callback choices.

The current numbered-text fallback remains available, so old clients and failed interactive setup do not block the scenarios.

## Scope

Supported:

- `/traektoria` opens the main scenario menu;
- `/traektoria text`, `/traektoria quiz`, `/traektoria deep`, `/traektoria wide`, and `/traektoria help` start the same flows as Telegram commands;
- Mattermost inline choices are sent as interactive message attachments when `MATTERMOST_ACTION_SECRET` is configured;
- button callbacks map to the existing Telegram-style `callback_data`;
- direct messages work without mentioning the bot;
- channel messages still require mention unless the user invokes the slash command;
- channel replies stay in the current thread when `MATTERMOST_REPLY_MODE=thread`;
- PDF delivery continues through Mattermost file upload when the bot account has permission.

Out of scope:

- replacing the existing scenario engine;
- building a Mattermost plugin;
- registering slash commands automatically through Mattermost admin APIs;
- adding complex interactive dialogs/forms.

## Configuration

Runtime secrets:

```env
MATTERMOST_TOKEN=<bot-account-token>
MATTERMOST_ACTION_SECRET=<random-secret-for-button-callbacks>
MATTERMOST_SLASH_TOKEN=<token-from-the-/traektoria-slash-command>
```

Public callback paths:

```env
MATTERMOST_ACTION_PATH=/mattermost/actions
MATTERMOST_SLASH_PATH=/mattermost/slash
```

`MATTERMOST_TOKEN` is the bot account token used for REST API and WebSocket access.

`MATTERMOST_ACTION_SECRET` is generated by us and embedded into each interactive button's confidential `context`. The callback endpoint rejects button requests when the secret is missing or invalid.

`MATTERMOST_SLASH_TOKEN` is copied from the Mattermost custom slash command setup screen. The slash endpoint rejects requests whose `token` field does not match.

## Mattermost Setup

Create one custom slash command in Mattermost:

- trigger word: `traektoria`;
- request URL: `https://bot.traektoria51.ru/mattermost/slash`;
- request method: `POST`;
- autocomplete: enabled;
- autocomplete hint: `[text|quiz|deep|wide|help]`;
- autocomplete description: `Открыть меню бота «Траектория талантов»`;
- response username/icon: optional, because the bot will post follow-up messages through the bot account.

No separate Mattermost integration token is required for interactive message buttons. Button authentication is handled by `MATTERMOST_ACTION_SECRET` inside action context.

## Data Flow

### Slash Command

1. User invokes `/traektoria` or `/traektoria <mode>`.
2. Mattermost posts form data to `MATTERMOST_SLASH_PATH`.
3. The bot validates `token === MATTERMOST_SLASH_TOKEN`.
4. The request is converted to a Mattermost target using `user_id`, `channel_id`, and team/channel metadata from the slash payload.
5. The selected command is routed to the existing `startScenarioFromCommand` path:
   - no argument: `/start`;
   - `text`: `/text`;
   - `quiz`: `/quiz`;
   - `deep`: `/deep`;
   - `wide`: `/wide`;
   - `help`: `/help`.
6. The HTTP response is a compact ephemeral acknowledgement. Real scenario messages are sent through the bot account so the same sender and file-upload behavior are used everywhere.

### Interactive Buttons

1. `sendMessage` receives a Telegram-style inline keyboard.
2. Mattermost transport creates a post with message attachments and actions.
3. Each action carries:
   - label;
   - callback data;
   - originating user id;
   - channel id;
   - root id;
   - `MATTERMOST_ACTION_SECRET`.
4. Mattermost posts action payloads to `MATTERMOST_ACTION_PATH`.
5. The bot validates the secret and reconstructs the Mattermost target.
6. The callback data is passed to the existing `handleCallback`.
7. The endpoint returns an ephemeral acknowledgement or an error payload for Mattermost.

## Target Identity

Direct message session key stays:

```text
mattermost:dm:<user_id>
```

Channel session key stays:

```text
mattermost:channel:<channel_id>:user:<user_id>
```

This keeps a user's direct conversation separate from channel-thread conversations.

For slash commands in channels, the reconstructed target should use the slash command's `channel_id` and `user_id`. If the command is invoked inside an existing thread and Mattermost includes a root/post id, replies should use that root id; otherwise replies are posted to the channel.

## Error Handling

- Missing `MATTERMOST_ACTION_SECRET` disables interactive attachments and keeps numbered text choices.
- Missing `MATTERMOST_SLASH_TOKEN` disables slash handling with an explicit server log warning.
- Invalid action secret returns `401`.
- Invalid slash token returns `401`.
- Unknown slash subcommand returns a short ephemeral help message.
- If interactive callback processing fails, the endpoint returns a Mattermost error response without crashing other transports.
- File upload failures keep the existing fallback message.

## Testing

Automated tests:

- Mattermost message formatting produces interactive attachments when action config is present;
- text-number fallback remains unchanged when action config is absent;
- slash command parser maps empty/text/quiz/deep/wide/help to existing commands;
- invalid slash/action tokens are rejected;
- action payloads reconstruct Mattermost direct and channel targets correctly;
- existing Telegram, MAX, web-chat, scenario, and target tests remain green.

Manual acceptance:

- `/traektoria` opens the main menu in a direct message;
- `/traektoria quiz` starts scenario 2;
- clicking each main-menu button starts the same scenario as Telegram;
- multi-select steps in scenario 2 can be clicked and continued;
- channel mention still works in thread mode;
- `/health` shows Mattermost connected/authenticated after production restart.

## Rollout

1. Add code and tests.
2. Add non-secret config examples.
3. Update production `/opt/telegram-bot/.env` on Beget with:
   - `MATTERMOST_ENABLED=true`;
   - `MATTERMOST_URL`;
   - `MATTERMOST_TOKEN`;
   - `MATTERMOST_ACTION_SECRET`;
   - `MATTERMOST_SLASH_TOKEN`;
   - action/slash paths if defaults are not used.
4. Restart `traektoria51-bot`.
5. Check `https://bot.traektoria51.ru/health` and `journalctl -u traektoria51-bot -n 100 --no-pager`.
6. Manually test Mattermost direct message and channel mention.
