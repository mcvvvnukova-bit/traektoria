# Классификация тем программ

Статус: рабочее описание от 2026-06-14.

Документ описывает слой нормализации и классификации тем, которые извлекаются из учебно-тематических и календарно-тематических планов программ PFDO.

## Зачем это нужно

Сценарий углубленной траектории должен понимать, что ребенок уже проходил, и подбирать продолжение не только по направленности программы, но и по содержанию занятий.

Классификатор превращает сырые строки тем в устойчивую иерархию:

```text
record_type -> domain -> category
```

Примеры:

- `content -> engineering -> robotics` для технических программ;
- `content -> tourism_skills -> hiking_routes_technique` для туристско-краеведческих программ;
- `service -> service -> assessment` для аттестации и контроля;
- `noise -> noise -> ocr_noise` для OCR-шума и табличных фрагментов.

## Где находится код

Основная логика:

- `services/program-topic-extractor/src/classification/technical-topic-classifier.js`

Скрипт построения аналитического слоя:

- `scripts/build-pfdo-topic-analytics.js`

Тесты:

- `test/topic-classifier.test.js`

## Как запустить

Перед запуском в `pfdo_51_mirror` должны быть загружены программы, документы и строки календарных тем.

```bash
node scripts/build-pfdo-topic-analytics.js
```

Скрипт очищает и пересобирает производные таблицы:

- `pfdo_program_topic_normalizations`;
- `pfdo_program_topic_aggregates`;
- `pfdo_program_topic_classifications`;
- `pfdo_program_topic_review_queue`.

## Входные данные

Классификатор работает с агрегированными темами из `pfdo_program_calendar_topics`.

Для классификации важны:

- `topic_name` — исходная тема;
- `section_title` — раздел плана;
- `program_id` и название программы;
- `directionName` — направленность программы;
- часы и тип занятия, если они извлечены парсером.

## Типы записей

| Тип | Что означает |
| --- | --- |
| `content` | Предметная тема, которую изучает ребенок |
| `service` | Служебная строка: контроль, аттестация, расписание, методика, оборудование |
| `noise` | OCR-шум, фрагменты оглавления, кабинеты, слишком общие строки |

`unknown_content` остается предметной категорией. Это важно: неопределенная тема не смешивается с мусором и попадает в ручную проверку.

## Поддерживаемые направленности

### Техническая

Техническая таксономия покрывает:

- программирование и алгоритмы;
- робототехнику и LEGO;
- электронику и Arduino;
- веб-разработку;
- игры и приложения;
- 3D-моделирование и печать;
- САПР, черчение и инженерную графику;
- техническое моделирование;
- цифровой дизайн, медиа и анимацию;
- данные, сети и цифровую безопасность;
- математику, логику и инженерно-научные основы;
- ПДД, транспорт и первую помощь;
- проектную и исследовательскую деятельность.

### Туристско-краеведческая

Туристско-краеведческая таксономия покрывает:

- основы туризма и туристский быт;
- снаряжение и туристские узлы;
- походы, маршруты, технику и тактику;
- безопасность и выживание в природной среде;
- первую помощь и походную медицину;
- карты, компас, топографию и спортивное ориентирование;
- краеведение, региональную историю и военную память;
- природу, географию и экологию края;
- этнографию, культуру и традиции;
- музейное дело, фонды и экспозиции;
- экскурсоведение, экскурсионные маршруты и путеводители;
- туристские соревнования и физическую подготовку;
- юнармию, строевую подготовку, ГО и спасательные работы.

Направленность выбирает профиль правил. Например, `Компас 3D` в технической программе относится к инженерной графике, а `Компас и карта` в туристско-краеведческой программе относится к топографии.

## Как работает классификация

1. Нормализация очищает тему от номеров, табличных хвостов, типовых префиксов и вариантов написания.
2. По направленности программы выбирается профиль правил.
3. Шум и служебные строки отделяются до предметной классификации.
4. Для технического профиля применяются проверенные golden labels из `pfdo_topic_classifier_golden_labels`.
5. Для предметных тем считается rule-based score по теме, разделу и названию программы.
6. Результат сохраняется вместе с confidence, top categories и matched rules.

Golden labels сейчас ограничены техническим профилем, потому что исходная ручная разметка была собрана для технической направленности.

## Ручная проверка

В `pfdo_program_topic_review_queue` попадают темы, если:

- категория осталась `unknown_content`;
- confidence ниже `0.6`;
- частая неизвестная тема встретилась несколько раз;
- строка выглядит как шум, но содержит много часов.

Очередь нужна для последующего пополнения правил или golden labels.

## Проверка изменений

Минимальная проверка классификатора:

```bash
node --test test/topic-classifier.test.js
```

Полный набор тестов проекта:

```bash
npm test
```

После изменений в техническом классификаторе можно дополнительно оценить качество на golden labels:

```bash
node scripts/evaluate-pfdo-topic-classifier.js
```

Эта оценка зависит от доступности локальной базы `pfdo_51_mirror`.
